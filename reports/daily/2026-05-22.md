#### 背景

Iceberg v4 格式规范（spec）引入了新的数据模型，但之前 manifest 文件（记录数据文件元数据的清单）写入时仍只支持 Avro 格式。本 PR 使 manifest 的写入层能够支持 **Parquet 和 Avro 两种格式**，并完整集成 v4 spec 的原生加密（Native Encryption）机制。

#### 核心改动

**1. `ManifestWriter.java` — 引入格式感知逻辑**

```java
// 新增字段：记录当前写入的文件格式
private final FileFormat format;

// 构造时根据文件扩展名推断格式
this.format = FileFormat.fromFileName(file.encryptingOutputFile().location());
this.file = outputFile(file);

// 关键新方法：Parquet 格式使用原生加密而非全文件加密
private OutputFile outputFile(EncryptedOutputFile encryptedFile) {
    if (format == FileFormat.PARQUET
        && encryptedFile instanceof NativeEncryptionOutputFile nativeFile) {
      return nativeFile;  // 使用 NativeEncryptionOutputFile，而非包裹后的 encryptingOutputFile
    }
    return encryptedFile.encryptingOutputFile();
}
```

**2. `V4Metadata.java` — v4 spec manifest 的 schema 更新**

扩展了 `V4Metadata` 以支持 Parquet 格式的 manifest 写入，包括 data file 和 delete file 的 Parquet schema 定义。

**3. Benchmark 重构**

将原有的 `ManifestReadBenchmark` 和 `ManifestWriteBenchmark` 合并重构为：
- `ManifestBenchmark.java`（主入口）
- `ManifestBenchmarkUtil.java`（共享工具类）
- `ManifestCompressionBenchmark.java`（新增压缩专项 benchmark）

#### 影响

- ✅ v4 格式表的 manifest 现在可以使用 Parquet 格式存储，利用 Parquet 的列式压缩和原生加密优势
- ✅ 为后续 Parquet manifest 性能优化奠定基础
- ✅ 全面覆盖测试：`TestManifestWriterVersions` 测试用例扩展至 125 行
